# xrayng UI 页面与组件规则

这个文件聚焦页面级与组件级的执行规则，供实现时对照。

## 现有组件家族

- section card：设置页、更多页、关于页、路由页、日志页的大面板容器
- settings row / more action row：约 68dp 行高、20dp 图标、14dp 图标文字间距、轻分隔
- home list card：首页节点卡片，圆角更大，内部用协议胶囊、延迟标签、更多按钮组织
- home dock：首页底部连接区，与首页卡片同语言
- summary panel：路由页顶部统计面板，属于 section card 内的次级面
- subtle icon button：约 36dp 弱底板次级操作按钮
- chip / badge：协议、订阅、状态、延迟等轻量信息标签
- preference intro / grouped preference row：设置页顶部说明和 Preference 行项，需拟合 settings row / more action row 的节奏

## 稳定产出流程

1. 先归类页面：设置页、列表页、工具页、首页、编辑页
2. 读取当前页、至少两个同级页，以及共享 `colors/styles/drawable`
3. 如果是首页任务，再读 `references/home-main-surface-system.md`
4. 写出简短判断：当前页属于现有哪种页面语言，问题是“不统一”还是“本身不好看”
5. 决定改动层级：只收颜色和圆角、调整结构、还是重做整个页面
6. 能先收共享资源时先收共享资源；共享资源解决不了再改单页结构
7. 修改后对照同级页面复看，确认没有长成另一套产品

## 页面级限制

### 设置式页面

- 背景保持冷白灰页面
- 默认用 section card 包内容，不拆成多张离散小卡
- 行内图标保持单色线性，不加额外底板
- section 间距、标题、分隔线、行高必须和已有页面一致
- “更多”页面是设置式页面高优先级参考
- 设置页选项背景若不统一，优先参考“更多”页入口行背景与分隔结构
- PreferenceFragment 外层分组背景与行内点击背景要分层：外层负责圆角与底色，行内负责 click/ripple
- 设置页顶部说明默认应存在，语气与 About / Routing / Subscription 子页一致
- 内容宽度优先使用 `settings_page_max_width`
- 分组标题允许使用极轻度冷灰强化导航，但正文和说明仍保持中性文字，不把整页做成彩色面板
- section card、settings row、more action row 的默认背景统一为纯白色

### 首页 / 列表页

- 保留首页的留白、胶囊、dock、协议与延迟标签语言
- 首页圆角可略大于设置页，但不能继续无限放大
- 工具按钮与现有顶部入口、列表更多按钮同一尺寸体系
- 底部连接区必须视为首页结构的一部分
- 首页背景统一为冷白灰，首页卡片、dock、次级容器统一为纯白，不保留暖色、蓝色或黄绿色装饰底
- 首页卡片、选中态、协议 chip 和 dock 主按钮优先使用首页专属 token；但高级感优先时，accent 只留给必要按钮和状态，不做气泡光斑和高光装饰
- 首页 dock 必须保持纯白卡片、冷灰分隔线和近黑主文字，不把灰底、暖灰或偏色线条带回首页
- 首页 dock 的信息表达优先复用现有摘要位；连接后的出口 IP / 地区等反馈默认写进这一行，不轻易再加次级按钮或额外信息行
- 首页列表卡片优先使用“外层 shell + 内层 inset panel”的双层结构，层次依靠阴影和细高光，不再依赖灰色描边
- 首页选中态必须可见，且优先同时作用在外层 shell 和内层 panel；选中色使用冷灰，不使用彩色块
- 首页列表更多按钮、顶栏动作按钮保持透明点击面，不做灰色背景底板
- 延迟 badge 固定按 `3 位数字 + ms` 预留宽度，避免不同数值导致宽度跳动
- 安全 badge 中 `TLS` 和 `REALITY` 都视为安全，使用绿色圆形背景加黑色盾牌；非安全传输使用灰色背景
- 首页 dock 不使用描边，允许极弱地图线稿装饰，但装饰不能影响正文和主按钮识别
- 首页 dock 动态摘要一律保持中性文字，不用绿 / 黄 / 红字体直接表达状态
- 首页文字粗细优先跟随系统 font weight 设置；首页震动反馈只保留底部启动 / 停止服务

### 工具页 / 日志页 / 路由页

- 仍属于设置页大语言，不单独做第三套视觉
- 容器、圆角、背景深浅、次级按钮复用现有写法
- 列表项默认收在统一面板里，不裸贴页面底
- 背景统一冷白灰，主容器和内部卡片优先纯白

## Preference 页面专项规则

- PreferenceCategory 标题、顶部说明、设置行不保留系统默认密度和组件观感
- 设置行标题接近 `15sp` 到 `16sp`，副标题接近 `12sp`
- Preference 行分隔线、图标大小、文字起始位与 `MoreAction.Row` 对齐
- 自定义 adapter 分组时显式排除顶部 intro / 非行项，避免错误套用首尾圆角
- 外层背景与行背景不要同时带完整 ripple，避免双层反馈

## 浮动面板 / 菜单专项规则

- 适用于 overflow menu、上下文菜单、底部更多菜单的二级面板、带子菜单浮层、右键式面板、轻量设置弹层
- 面板背景默认纯白 `#FFFFFF`，强调与底层内容隔离
- 毛玻璃默认理解为“白底 + 轻 blur + 连续高光”，不是灰色半透明块
- 毛玻璃底色仍读作白色，可用接近 `#FFFFFF` 的低透明白或白到极浅灰渐变
- 高光必须连续过渡，避免中上部固定高度 sheen 条带
- 边缘只允许极弱外轮廓与内高光，禁止明显实线边框
- 层级靠轻 blur、顶部连续高光、极弱轮廓与柔和阴影建立
- 主要文本颜色接近 `#121212` 到 `#1F1F1F`
- 次要信息统一中灰，接近 `#8B8F97` 到 `#9AA0A6`
- hover / active / selected 用浅灰圆角背景表达，接近 `#F0F1F3`
- 面板外圆角 `12dp` 到 `16dp`
- 面板内可点击项圆角约 `8dp`
