# xrayng APP UI 参考规范

这个文件用于指导 agent 设计标准化、产品化的 App UI，直接给出可落地的页面规范、配色关系和修改边界。

## 零、当前产品风格总览

在当前项目里，agent 不应把 UI 理解成“每个页面都可以各做各的”，而应先把现有产品语言看清楚：

- 当前产品主基调是暖白背景、浅灰面板、近黑文字、克制蓝色强调
- 设置页、更多页、关于页、路由页属于同一类：整组 section panel、轻分隔、行内图标、不做图标底板
- 首页属于另一种更活跃的表达：更大留白、更圆卡片、更明显的胶囊、顶部栏目和底部 dock 更强
- 虽然首页更有结构感，但它和设置页仍共享相同的背景灰阶、文字气质和按钮克制度
- 整个产品不是靠厚阴影和强描边建立层次，而是靠背景深浅、圆角、间距、轻量分隔和少量高对比焦点

### 设置式页面当前观感

- 页面背景偏雾白，内容通常居中并限制最大宽度
- 一级内容按逻辑收进整组面板，不拆成很多独立卡片
- 行高稳定，图标尺寸稳定，图标与标题间距稳定
- 主次文字关系温和，说明文字较短，不靠大段说明堆层级
- 蓝色通常只在主操作、可切换控件、关键状态中出现

### 首页当前观感

- 首页比设置页更强调留白和节奏感
- 列表项是浅灰卡片，圆角比设置页更大一些，但仍属于同一产品
- 协议标签、延迟标签、顶部分组、底部连接区都倾向胶囊化
- 顶栏和底部连接区是首页识别度的关键区域，改动时必须和列表项一起考虑
- 首页按钮虽然更显眼，但仍然保持克制，不走大面积彩色铺底

### 对 agent 的直接要求

- 任何单页改版前，先判断它属于“设置式页面”还是“首页 / 列表式页面”
- 任何统一类任务，先看共享颜色、共享圆角、共享按钮和共享列表写法，再动单页
- 如果改完一个页面后，它和相邻页面不像同一个产品，说明方向错了

### 当前代码里已经固定下来的视觉规律

- 设置页、更多页、关于页、路由页通常有最大宽度约束，内容在中间，不直接铺满页面
- 设置行和更多入口的主行高基本落在 `68dp` 到 `72dp`
- 设置行图标多数是 `20dp`，图标与标题间距约 `14dp`
- 设置页分组通常是 `18dp` 圆角
- 首页列表卡片与底部 dock 的圆角明显更大，但仍维持暖灰浅面板关系
- 首页按钮、tabs、协议标签、延迟标签普遍使用胶囊形态
- 路由页会在 section card 内再嵌 summary panel 和规则卡片，不是裸列表
- 日志页是“页面背景 + 主面板 + 面板内次级项”的层级，不是直接把日志 item 贴在页面底

这些规律应被视为现有标准，不应轻易打破。

## 零点五、严格限制 agent 的 UI 输出

为了确保统一，这个项目里的 agent 输出应被限制在现有产品语言之内：

- 允许：收颜色、收圆角、收间距、统一按钮、统一行高、统一列表项、统一表单
- 允许：在现有页面语言内部增强结构感，比如更清晰的 section、轻量 badge、统一空状态
- 允许：首页在保留当前语言的前提下做更强的留白、胶囊和顶部 / 底部联动
- 不允许：新增一整块 hero header、营销卡、大插画头图、大面积彩色背景
- 不允许：把设置页做成强装饰首页风格
- 不允许：把首页做成普通工具页或传统设置页
- 不允许：引入当前项目里不存在的新按钮体系、新卡片体系、新描边体系
- 不允许：同页混用多套圆角、多套按钮尺寸、多套文字层级
- 不允许：只把某一页做得更花，但和周围页面明显脱节

### 输出前必须自检

- 这次改动是否仍然属于当前产品的两类页面语言之一
- 这次改动是否优先复用了共享颜色、共享卡片、共享按钮和共享行结构
- 这次改动是否减少了风格分裂，而不是制造了新的例外
- 这次改动如果截图给用户看，是否会被误认为另一个 App

## 一、参考图对应的设置页规格

### 背景关系

- 参考图属于“亮页面托住浅灰卡片”，页面亮度高于卡片
- 页面背景：`#F7F6F2`
- 一级卡片背景：`#F1EFEC`
- 分组面板背景：`#ECE9E4`
- 分隔线：`#DFDAD3` 到 `#E6E1DA`
- 主文字：`#1F1F1F`
- 次文字：`#786F65`
- 主 CTA 蓝色：`#2785FF`
- 主 CTA 浅色容器：`#E4EDF8`
- 危险文字 / 图标：`#E15B5B`

### 夜间背景关系

- 默认夜间页面背景：`#11151B`
- 默认夜间一级卡片背景：`#1A2027`
- 默认夜间分组面板背景：`#232B34`
- 默认夜间分隔线 / 轮廓：`#303945` 到 `#36414D`
- 默认夜间主文字：`#FFFFFF`
- 默认夜间次文字：`#A4AFBC`

### 纯黑夜间模式变体

- 当用户明确要求“夜间背景纯黑”时，页面背景允许直接落到 `#000000`
- 纯黑背景启用后，一级卡片必须同步压暗，优先接近 `#101317` 到 `#12161A`
- 纯黑背景启用后，弱层级面板 / 次级容器必须同步压暗，优先接近 `#171B20` 到 `#171C22`
- 首页纯黑背景下，首页主卡片优先接近 `#0E1115` 到 `#12161A`
- 首页纯黑背景下，次级胶囊 / tabs / 小按钮优先接近 `#171B20`
- 纯黑背景下，分隔线 / 轮廓要再弱一档，优先接近 `#232931` 到 `#2E343D`
- 纯黑背景下，不允许继续保留偏亮的深灰蓝卡片，否则会出现“黑底上漂浮灰块”的割裂感
- 纯黑背景下，顶栏和底部 dock 的夜间渐变也必须同步切到黑色过渡，不能继续沿用 `#11151B` 系列渐变
- 纯黑背景下，主按钮仍可保持高对比，但卡片内状态 badge、输入框底和次级操作面板要整体收暗，避免局部发灰
- 纯黑背景优先用于首页、工具页、日志页这类强调沉浸感的页面；若是设置式页面，没有明确要求时仍优先使用深灰蓝夜间底

### 圆角与尺寸

- 设置页分组圆角：`18dp`
- 设置行背景圆角：`18dp`

- 小按钮圆角：`16dp`

- 行左右内边距：`16dp`
- 行上下内边距：`12dp`
- 图标大小：`20dp`

### 文本层级

- 页面标题：`20sp` 左右，粗体

- 行标题：`15sp` 到 `16sp`，粗体
- 行副标题：`12sp` 到 `13sp`
- 版本信息：`11sp` 到 `12sp`

## 二、设置式页面必须满足

- 页面头部轻量
- 不用大卡片头
- 不给图标加底板
- 只有主 CTA 使用 `#2785FF`
- 普通行、分组面板、顶部入口不使用纯蓝底
- 不堆很多独立小白卡片
- 每个分组看起来像一个完整面板
- 危险操作只有真实存在时才展示

## 三、列表页必须满足

- 可以保留卡片结构
- 但圆角不能比设置页更大
- 操作按钮不能像另外一套组件
- 背景灰阶要与设置页一致
- 列表卡片不要纯白，必须落在统一暖灰卡片体系内
- 夜间列表卡片不要做成纯黑底上浮一层亮块

## 四、首页黑字主题规格

- 首页背景：`#FBFAF7`
- 首页主卡片：`#F2EFEB` 到 `#F4F1EC`
- 首页次级胶囊 / tabs / 小按钮：`#EEEAE3`
- 首页主文字：`#121212`
- 首页次文字：`#8C877F`
- 首页描边：`#E5E0D8`
- 首页主按钮：近黑底 `#121212`，文字 / 图标使用 `#FFFFFF`
- 首页列表选中：优先用黑色小指示、黑色描边或轻微提亮，不再用蓝色选中块
- 首页空状态：弱图形 + 留白 + 2 到 3 个浅灰胶囊操作，不要大段说明
- 首页底部连接区：优先做成整块浅灰 dock，黑色按钮作为焦点
- 首页顶部 tabs：外层容器和底部 dock 使用同一组浅灰背景关系，选中态是容器内再包一层更深半档的胶囊
- 首页工具按钮：顶部更多入口、加号按钮、列表更多按钮统一使用浅灰胶囊按钮，不使用透明悬浮图标
- 首页搜索态：搜索输入框只保留一层浅灰输入胶囊，输入法弹出后键盘上方区域必须是页面本色，不能留下灰块遮挡
- 当用户要求去掉灰框时：首页顶部搜索、左侧 app 名称、加号和更多入口都改为透明点击面，反馈只靠轻触动画和无底板 ripple
- 当 tabs 并入顶部栏目时：优先做成文字 tabs + 黑色底部指示线，不再额外包浅灰大容器
- 当 tabs 与顶栏左右入口共处一行时：左右按钮和 app 名称要主动缩小，给中间分组让出空间，图标 / 文案必须居中对齐
- 首页顶栏背景可以改成从雾白到透明的纵向渐变，让列表与顶栏过渡更软，同时保留黑字主题的轻感
- 使用顶栏渐变时：`AppBarLayout` 承担背景层，`Toolbar` 自身保持透明，避免运行时代码把渐变覆盖掉
- 如果需要更接近全面屏，优先降低顶栏和底部 dock 的基础高度，并收掉它们与屏幕边缘之间多余的雾白留白
- 真正的首页 edge-to-edge 需要同时满足三件事：`WindowCompat.setDecorFitsSystemWindows(window, false)`、去掉布局里残留的 `fitsSystemWindows="true"`、手动把状态栏 inset 只加到顶栏容器而不是整页内容
- tabs 常驻时，滚动联动应改成“背景透明度变化”而不是“组件显隐变化”
- 顶栏透明联动：优先改 `AppBarLayout` 的背景 drawable alpha；底部 dock 透明联动：优先改 card background color alpha
- 真正的首页透明感通常要求顶栏覆盖在列表上方；如果顶栏还处在普通文档流里，就算背景透明，视觉上也仍会留出一整块空白
- 顶栏放进覆盖层后，绘制顺序要在列表之后，否则会被内容层级冲掉，透明效果也会显得失真
- 底部 dock 的列表底 padding 和自身底 margin 都要轻量处理；如果直接把 system bar inset 全量叠加，底部会出现明显空白，看起来像“白底托住卡片”
- 首页如果同时有顶栏、dock、搜索、IME、分组滚动联动，优先建立 `AppChromeStateReducer -> TopBarRenderer / BottomBarRenderer` 的链路，不要继续在 `Activity` 里分散直写 alpha
- “只在滑动时透明”的推荐语义是：`IDLE = STABLE`、`DRAGGING/SETTLING = SCROLLING_IMMERSIVE`、`SEARCH/IME = override state`、`EMPTY = EMPTY_STABLE`
- 顶栏 / 底栏透明度优先使用有限档位，例如 `solid / soft / floating`，避免连续 progress 造成高频无效 render
- Chrome 状态调试如果需要日志，只记录状态枚举、滚动相位、匿名 group 标识，不记录搜索词和节点信息
- 首页折叠态搜索按钮优先和右上角工具按钮统一成 `36dp` 量级的小圆按钮
- 首页左上如果不保留 app 名称：默认只放菜单按钮，尺寸可以比右侧工具按钮略大一档
- 首页右侧搜索、加号、更多按钮要统一图标视觉大小和相邻间距，避免出现系统默认搜索图标偏大、偏疏的问题
- 搜索和加号之间如果看起来过散，通常先处理折叠态搜索根视图的保留宽度，再处理加号的起始 margin
- 在不改搜索展开逻辑的前提下，可以把加号和更多合并为一个 action cell，减少右侧被 `ActionMenuView` 切成多个单元格的分散感
- 如果要进一步合并成单按钮，优先保留一个 toolbar hub 入口，通过底部菜单承载搜索 / 添加 / 更多动作
- hub 入口本身要避免继续长成普通“更多”图标，优先做成搜索主图标配纵向双点或提示点的小胶囊视觉
- 如果组合视觉和系统菜单残留同时存在，优先去掉残留搜索按钮和 overflow，再把 hub 收成单一图标，保证右侧只读出一个入口
- 子页面统一的优先级：先统一 `background -> section card -> form field -> icon action button` 四层基础界面关系，再改具体布局
- 设置页 / 工具页 / 资产页 / 日志页：优先使用居中的 section panel、18dp 圆角、低对比面板分层和 36dp 弱底板次级操作按钮
- 编辑页：背景统一为雾白页面，内容区优先分成 section card；输入框和 spinner 使用更浅一档 field 底色，不再直接贴在页面背景上
- 服务器协议编辑页：基础地址卡之外的协议字段优先继续分组进同层级 section card，避免出现“一个卡片 + 一串裸字段”的旧式结构
- 协议标签：优先近黑底白字；延迟标签：根据状态使用浅绿 / 浅米 / 浅红底，文字对比必须明显
- 首页启停按钮：默认纯黑圆按钮，进入 loading 后只保留圆形进度层，不允许看到矩形背景或裁切痕迹
- 如果首页夜间模式改成纯黑底，首页的 `color_home_background`、`bg_home_top_bar_gradient`、`bg_home_bottom_underlay_gradient` 必须成组更新，不能只改单个颜色资源

## 五、禁止项

- 禁止超过 `18dp` 的设置页分组圆角
- 禁止出现圆形图标底板
- 禁止重描边、重阴影
- 禁止冷白背景上叠很多纯白卡片
- 禁止为了“占位”增加不存在的功能入口

## 五点五、浮动面板 / 子菜单专项规格

- 这组规格适用于所有浮动菜单、子菜单、context menu、overflow panel 和轻量操作浮层
- 面板主体背景固定为纯白 `#FFFFFF`，不要混入暖灰主卡片色
- 如果需要毛玻璃观感，仍然以白色面板为主，只允许叠加轻 blur、连续高光和极弱轮廓，不把主体做成灰色雾面块
- 毛玻璃底色优先控制在白到极浅灰之间，最终读感必须仍是白色面板
- 毛玻璃高光必须连续覆盖整块面板，优先全高渐变或整块 foreground overlay；禁止使用固定高度的独立高光带
- 毛玻璃边缘只允许极弱描边和内侧高光，不允许读出明确硬边框
- 如果实现用了多层叠加，必须避免出现横向分层线、局部亮暗断层和错误参与布局测量的 overlay
- 主文字固定使用高对比近黑，优先 `#121212`
- 次要信息、未激活说明、toggle 轨道使用中灰，优先 `#8F949B`
- 行 hover / active / selected 统一通过浅灰圆角底块表达，背景优先 `#F0F1F3`，不要改文字主色
- 面板外圆角优先 `12dp` 到 `16dp`
- 面板内交互项的状态背景圆角优先 `8dp`
- 不使用明确实线边框；层级依赖白底、轻阴影和极弱轮廓，不能读成厚描边浮框
- 菜单项内边距优先上下 `8dp`、左右 `12dp`
- 左侧图标、正文、右侧控件之间保持固定 gap，所有左侧图标必须垂直对齐
- 分割线两端必须留白，不允许顶格切到容器边缘
- 有子菜单的项右侧必须显示右箭头
- 布尔项使用 toggle
- 单选项当前值右侧必须显示勾选
- 同一面板内优先使用单一的右侧状态语义：箭头对应子菜单、toggle 对应布尔值、勾选对应单选，不叠加重复提示

## 六、改版后的检查清单

- 页面背景是否统一
- 分组背景是否统一
- 圆角是否统一
- 分隔线是否足够轻
